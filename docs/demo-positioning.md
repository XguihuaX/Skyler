# Skyler Demo Positioning

> 本文档是 demo 视频和投递材料的文案依据。所有表述以本地代码真实状态为准:已落地能力可以明确展示;进行中能力必须标注;规划能力不能包装成已完成。

## 总体定位

Skyler 是一个**连续人格驱动的 Live2D 桌面 AI 陪伴角色系统**。

它不是给问答 AI 套上角色外观,而是围绕 Persona、状态系统、四层记忆、活动时间线和多源上下文组装,让角色持续积累状态,逐渐形成自己的节奏和生活感。

## 章节 0:开场

### 这一页展示什么

展示 Skyler 的第一印象:Live2D 桌面角色、启动动画、产品气质。

### 可以说的已落地能力

- Live2D 桌面角色形态。
- 启动动画。
- 主项目已具备主面板和桌宠小窗两种形态。

### 必须标进行中的能力

- 不需要在开场页展开进行中能力。

### 不能说的话

- 不要说 Skyler 已经是完整商业产品。
- 不要说所有平台都已验证;当前主要是 macOS Apple Silicon。

### 推荐视频文案

Skyler:连续人格驱动的 Live2D 桌面 AI 陪伴角色系统。

## 章节 1:总览

### 这一页展示什么

解释 Skyler 是什么,以及后续 demo 会展示哪些模块。

### 可以说的已落地能力

- 双形态 UI:全屏主面板 + 桌宠小窗。
- Live2D 渲染、模型管理、framing。
- 文本、语音、图片、文件当轮输入。
- Persona、状态、记忆、活动上下文、只读桌面感知、MCP 能力扩展。

### 必须标进行中的能力

- DailyAgent 完整 FSM。
- 上下文仲裁策略。
- 主动视觉感知。
- 图片/文件长期记忆沉淀。

### 不能说的话

- 不要说 Skyler 是完全离线系统;应说 local-first,provider 可替换。
- 不要把建设中能力写成已完成。

### 推荐视频文案

Skyler 不是普通聊天框。她有 Persona、状态、记忆和活动上下文,以 Live2D 角色形式常驻桌面,并通过 MCP 扩展工具能力。

## 章节 2:交互

### 这一页展示什么

展示用户如何与 Skyler 交互。当前视频主要展示语音链路,但文案可以说明文本、图片、文件作为当轮输入也已接入。

### 可以说的已落地能力

- 文本输入。
- 语音链路:VAD / ASR / LLM streaming / TTS / Live2D 表演。
- 图片作为当轮 image block 输入。
- 文件抽取文本后作为当轮上下文输入。

### 必须标进行中的能力

- 图片/文件持久化。
- 图片/文件进入长期记忆和后续召回。
- 更完整的多模态输出。

### 不能说的话

- 不要说图片/文件已经完整沉淀进长期记忆。
- 不要说视觉理解必然可用;图片理解取决于当前 active model 是否支持多模态。

### 推荐视频文案

语音不是事后朗读文字,而是一条实时链路:VAD 捕捉语音,ASR 转写意图,LLM 流式生成,TTS 输出角色声音,Live2D 同步表演。

## 章节 3:角色

### 这一页展示什么

展示 Persona、角色详情、card_type 和角色卡机制。

### 可以说的已落地能力

- 结构化 Persona。
- 多 variant。
- 角色详情 UI。
- Persona 编辑器。
- card_type 基础链路:社交型 / 助手型。
- 社交型偏日常交流、关系和情绪表达;助手型偏任务、工具能力和行为边界。

### 必须标进行中的能力

- 社交型 / 助手型运行时策略进一步分流。
- 多角色 Persona 内容完善。
- Persona v2 继续清理死字段和模板漂移。

### 不能说的话

- 不要说完整多角色生态已经完成。
- 不要说 card_type 已经驱动所有运行时行为。

### 推荐视频文案

角色不是一段单 prompt,而是由身份、语气、边界、样例、状态和 card_type 共同组成的结构化 Persona。

## 章节 4:活动

### 这一页展示什么

展示活动时间线、上下文连续性、主动触发雏形和 DailyAgent Stage 1。

### 可以说的已落地能力

- 活动时间线。
- active app / browser / document / URL content 等活动上下文。
- 活动 session 持久化。
- 今日活动 prompt 注入。
- 主动触发雏形。
- idle gate / throttle / daily cap / active conversation guard。
- DailyAgent Stage 1:日计划生成 + 当前活动 ticker + 状态写回。

### 必须标进行中的能力

- DailyAgent 完整 FSM。
- 多角色调度。
- 更完整的角色日常节奏系统。
- 上下文仲裁策略增强。

### 不能说的话

- 不要说 DailyAgent 已经完整完成。
- 不要说角色已经长期自治。

### 推荐视频文案

Skyler 不会每轮归零。她会保留近期活动、角色状态和用户上下文,让对话基于最近发生的事情继续。

## 章节 5:感知

### 这一页展示什么

展示 Perception / 桌面感知。重点不是“读屏”这个单点,而是 Skyler 如何从桌面环境获取结构化证据。

### 可以说的已落地能力

- 只读 AX / UI Tree 桌面感知。
- 读取前台 app、窗口标题、可见 UI 文本、浏览器 URL / 内容。
- 基于可读取证据回答当前屏幕状态。
- 小窗形态下支撑低打扰陪伴和上下文理解。

### 必须标进行中的能力

- 主动视觉感知。
- 小模型常驻监视变化。
- 判断是否需要截图式视觉理解。
- 截取当前窗口或局部屏幕。
- 调用 Qwen Plus / Flash 等支持图片输入的模型理解当前画面。
- 写操作确认门端到端验证。

### 不能说的话

- 不要说完整 VLM 屏幕视觉系统已完成。
- 不要把用户上传图片/文件与主动截屏视觉感知混为一谈。
- 不要展示或暗示自动写操作已经安全可用。

### 推荐视频文案

当前已落地的是只读 AX / UI Tree:Skyler 基于前台窗口的结构化证据理解你正在做什么。截图式视觉理解和常驻主动感知仍在建设中。

## 章节 6:能力

### 这一页展示什么

展示 CapabilityRegistry、MCP、provider 替换和安全边界。

### 可以说的已落地能力

- CapabilityRegistry。
- 内建 capability 自动注册给 ChatAgent。
- MCP client 接外部 tools。
- MCP server 暴露 Skyler 自身 capability。
- per-tool 开关。
- provider 可替换。
- confirm gate 框架。

### 必须标进行中的能力

- dangerous tools 端到端验证。
- 凭证治理。
- 更完整安全策略。

### 不能说的话

- 不要说所有危险工具已经绝对安全。
- 不要说所有外部 MCP server 都默认可用;真实能力取决于本地配置和凭证。

### 推荐视频文案

能力不是写死在聊天逻辑里。Skyler 用 CapabilityRegistry 统一管理内建能力和 MCP tools,并用 per-tool 开关和确认门治理危险操作。
