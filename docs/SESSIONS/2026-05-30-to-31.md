## 3 · 主线 2 · 凌晨真机 trial + Pit 1 sticky pause 真因

### 3.1 PM 真机 trial(2026-05-31 凌晨)

- `pgrep -fl mpv` 输出确认无 `--media-keys=yes`(arg fix 真应用)
- `nc -U /tmp/skyler_mpv.sock` socket query 拿 mpv state
- `curl` NCM CDN URL 实测 200 + 5MB(URL 真 work)
- `get_property "path"` 返完整 URL(file 已 load)
- `get_property "playback-time"` 一直返 `0.000000`(卡死)
- 手动 `set_property pause false` → 立刻出声

### 3.2 Pit 1 真根因

不是"mpv loadfile 后默认 paused"(我 repro A 实证 fresh spawn loadfile 后
pause=False · playback-time 正常 advance)。

是 **sticky pause 跨 loadfile**:singleton 此前若被 `pause()` 调过(LLM 误调
`netease_local(action='pause')` / 手测 IPC pause / 老 session 留下) · mpv 0.41
loadfile-replace 不重置 pause 属性 · 新文件 playback-time 卡 0。

Repro B 复刻成功:
```
1. loadfile + set pause=True (模拟之前的 pause)
2. confirm paused: True
3. loadfile NEW replace
4. pause 状态: 仍 True (sticky!)
5. playback-time t0: 0.023
6. playback-time t+0.6s: 0.023 (没 advance)
```

### 3.3 修法(包在 Commit 2 · 1 行)

`play()` 内 loadfile 之后立即:
```python
await self._send_command(["set_property", "pause", False])
```
idempotent(已 False 也无害)。

Smoke test 实证: pre-play pause=True → post-play pause=False → playback-time
advance 0.134s · 真修了。

## 4 · 主线 3 · Pit 2+3 · tool 路径混乱(凌晨现场)

### 4.1 现场

- 用户说"播放当前停止的音乐" → LLM 选 `media.play_pause` → MediaRemote 控了
  NCM 客户端播旧曲("月亮不会奔你而来"非 Reaching Light)
- 用户说"播放日推第一首" → LLM 选 `netease_web.daily_recommend` → mpv 后台播
  Reaching Light

两个 audio source(MediaRemote vs 内嵌 mpv)独立并存 · UX 不一致。

### 4.2 三套 cap 重叠矩阵

| 用户意图 | media | netease_web | netease_local |
|---|---|---|---|
| 暂停/继续 | `play_pause`(toggle) | — | `pause`/`resume` |
| 下一首 | `next_track` | — | `next_in_queue` |
| 上一首 | `previous_track` | — | (无) |
| 当前在播 | `now_playing` | — | `now_playing`(Patch D) |
| 音量 | `set_volume` | — | (无) |
| 放日推 | — | `daily_recommend` | — |
| 放某歌 | — | `play_song(keyword)` | — |
| 放歌单 | — | `play_playlist`+`by_id` | `play_playlist(id)` |

3 个重叠 action: `play_pause`/`pause` · `next_track`/`next_in_queue` · `now_playing`/
`now_playing`。

### 4.3 tool_addendum 现状(改前)

- 【音乐类】(L41-56) 说 `play_song` 一步 netease_web
- 【网易云本地 mpv】(L88-106) 说 `play_song` 两步 search+netease_local
- 两版矛盾 · LLM 看着反复横跳
- 【媒体控制】(L57-66) 无优先级声明 · LLM 看语义直觉 ("暂停" → media)
- Patch D 暴露的 netease_local.now_playing fallback 写在 L62-64 · 但 LLM 凌晨
  没走 fallback 路径(可能没读懂 / 可能直接编了答案)

### 4.4 修法(Commit 3 · `d712768`)

新结构 3 section · 替代旧 3:
- **【音频源优先级】** 必读 · 3 条规则:
  1. 会话用过 mpv-first → 后续控制首选 `netease_local.*`
  2. `not_running` 或从未走过 mpv → fallback `media.*`
  3. 用户明确说"系统级 / NCM 客户端" → 直接 `media.*`
- **【网易云本地 mpv 自动播放】** 整合 netease_web 发起 + netease_local 控制 +
  Patch D now_playing 首选 + like_current + mpv_error 防 LLM 编 PATH/重装 advice
- **【媒体控制】** fallback 定位 · 引用上面优先级

关键引导反转:`now_playing` 旧"先 media 后 fallback netease_local" → 新"首选
netease_local · 返 False 再 media" · media 看不见 mpv 是常态非错误。
